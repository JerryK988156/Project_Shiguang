# 拾光计划 v3.0 → v4.0 优化方案

> 基于 2026-07-20 当前版本，面向下一阶段的全面提升方案

---

## 一、当前版本概况

v3.0 已完成全部计划功能：用户注册/登录、目标管理、打卡系统、成就徽章、标签分类、统计分析（9 种图表）、JWT 鉴权、通知提醒、学习周报、AI 智能体 Agent、UI 艺术化加工。后端 33 个 API 端点，前端 11 个页面。

---

## 二、前端页面设计优化

### 2.1 加载体验优化

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 骨架屏 | 数据加载时无占位，页面闪动 | 使用 `el-skeleton` 为卡片、表格、图表添加骨架屏加载态 | 高 |
| 空状态引导 | 无数据时显示空白表格 | 添加 `el-empty` 插画 + 引导文案 + 快捷操作按钮（如"创建第一个目标"） | 高 |
| 图片懒加载 | 头像直接加载 | 添加 `v-lazy` 或 Intersection Observer 懒加载 | 低 |
| 请求防抖 | 快速切换状态筛选会连续请求 | 筛选输入添加 300ms debounce | 中 |

### 2.2 动效与交互增强

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 成就弹窗动效 | 弹窗出现较平淡 | 添加五彩纸屑（confetti）动画 + 音效提示 | 中 |
| 打卡成功反馈 | 仅有 message 提示 | 打卡按钮短暂变为勾选动画 + 卡片绿色脉冲 | 中 |
| 图表动画 | ECharts 默认动画 | 配置入场动画（逐帧展开、渐进渲染） | 低 |
| 拖拽排序 | 目标列表固定顺序 | 引入 `vuedraggable` 支持用户拖拽排序目标 | 低 |

### 2.3 页面级优化

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 目标详情抽屉 | 编辑需要跳转独立页面 | 在列表页添加右侧 `el-drawer` 快速查看/编辑目标详情 | 高 |
| 打卡快捷入口 | 需要导航到打卡页 | 看板卡片点击直接跳转今日打卡（携带目标ID预设） | 中 |
| 搜索与筛选增强 | 仅状态筛选 | 添加关键词搜索（目标标题、标签）+ 时间范围筛选 | 中 |
| 个人统计导出 | 仅打卡记录支持 CSV | 统计图表支持导出为 PNG 图片 | 低 |
| 页面面包屑 | 无导航层级提示 | 顶部栏添加 `el-breadcrumb` 显示当前位置 | 低 |

### 2.4 视觉细节打磨

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 头像裁剪 | 上传原图直接使用 | 集成 `vue-cropper` 支持上传前裁剪 | 中 |
| 图表主题统一 | ECharts 使用默认配色 | 自定义 ECharts 主题色板与全局 CSS 变量一致 | 中 |
| 弹窗确认美化 | 使用浏览器默认 confirm | 统一为 `el-message-box` 自定义样式确认弹窗 | 低 |

## 三、后端功能改进与提升

### 3.1 性能与稳定性

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| Redis 缓存 | 无缓存，每次实时查询 | 集成 Spring Cache + Redis，缓存统计数据（过期 5 分钟） | 高 |
| 接口限流 | 无限流保护 | 使用 `Bucket4j` 或 Guava RateLimiter 对高频接口限流（如 agent/chat） | 中 |
| SQL 优化 | 部分统计查询 N+1 | 使用 JOIN 查询 + 批量查询减少数据库往返 | 中 |
| 数据库连接池监控 | 无监控 | 添加 HikariCP 指标暴露 + 慢查询日志 | 低 |

### 3.2 安全增强

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 密码加密升级 | 使用简单 MD5 | 迁移至 BCrypt 或 Argon2，同时兼容旧密码 | 高 |
| Token 刷新机制 | JWT 永不过期 | 引入 access_token（短）+ refresh_token（长）双 token 机制 | 中 |
| 操作日志 | 无审计日志 | AOP 切面记录关键操作（登录、删除、状态修改）到 audit_log 表 | 中 |
| XSS 防护 | 前端隐式依赖 Vue 转义 | 后端添加输入过滤（Jsoup 或 Spring HtmlUtils） | 低 |

### 3.3 功能扩展

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| API 文档 | 无文档 | 集成 SpringDoc OpenAPI + Swagger UI 自动生成 API 文档 | 高 |
| 数据定时备份 | 无备份机制 | 使用 Spring `@Scheduled` + mysqldump 定时备份数据库 | 中 |
| 邮件通知 | 仅浏览器通知 | 集成 JavaMailSender，支持邮件提醒（周报、成就） | 中 |
| WebSocket 实时推送 | 无实时通信 | 集成 STOMP WebSocket，打卡时实时更新看板数据 | 低 |
| 文件上传通用化 | 仅头像上传使用 | 封装通用文件服务，支持打卡图片附件 | 低 |

### 3.4 Agent 智能体增强

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 流式输出 | 等待完整回复后一次性返回 | 使用 SSE（Server-Sent Events）实现打字机效果 | 高 |
| 对话历史持久化 | 刷新后历史丢失 | 新建 `agent_message` 表，按用户/会话存储对话历史 | 中 |
| 多轮上下文记忆 | 每轮发送完整历史 | 让 LLM 自行维护摘要 + 滑动窗口减少 token 消耗 | 中 |
| 推荐型建议 | 被动响应 | Agent 主动分析用户数据，给出学习建议（如"你的英语目标快到期了"） | 低 |

## 四、工程化提升

| 优化项 | 现状 | 方案 | 优先级 |
|--------|------|------|--------|
| 单元测试 | 零测试覆盖 | 后端添加 JUnit5 + Mockito 单元测试，前端添加 Vitest 组件测试 | 高 |
| CI/CD | 手动构建部署 | 配置 GitHub Actions 自动构建 + 测试 | 中 |
| Docker 化 | 无容器化 | 编写 Dockerfile + docker-compose.yml 一键部署 | 中 |
| 代码规范 | 无 lint 配置 | 后端配置 Checkstyle，前端配置 ESLint + Prettier | 中 |
| 日志增强 | 仅有 MyBatis SQL 日志 | 统一使用 SLF4J + 结构化日志，便于问题排查 | 低 |

## 五、优先级排序建议

### 第一阶段（短期，1-2 周）：高优先级项

1. **骨架屏 + 空状态** —— 直接影响用户感知的加载流畅度
2. **密码加密升级（BCrypt）** —— 安全基础
3. **API 文档（Swagger）** —— 便于后续协作和维护
4. **Redis 缓存** —— 提升高频统计查询性能
5. **目标详情抽屉** —— 减少页面跳转，提升操作效率

### 第二阶段（中期，2-3 周）：中优先级项

6. **Agent 流式输出（SSE）** —— 大幅提升对话体验
7. **单元测试建设** —— 保障代码质量
8. **Docker 化** —— 标准化部署
9. **搜索筛选增强 + 请求防抖** —— 列表页体验升级
10. **成就弹窗动效 + 打卡成功反馈** —— 情感化设计

### 第三阶段（长期，1-2 月）：低优先级项

11. WebSocket 实时推送、文件上传通用化等
12. PWA、离线访问、邮件通知
13. 社交功能、学习建议

## 六、技术债务清理建议

| 债务项 | 说明 |
|--------|------|
| `cookies.txt` | 根目录遗留文件，应加入 `.gitignore` 或删除 |
| `db.sql` vs `bootstrap.sql` | 两份 SQL 脚本内容可能不一致，建议统一 |
| 前端硬编码颜色残余 | 部分 Vue 组件中仍有直接写死的颜色值（如 `#fff`、`#111827`），应全部迁移到 CSS 变量 |
| 错误处理不统一 | 部分 catch 块为空或仅 `console.error`，应统一异常处理策略 |
