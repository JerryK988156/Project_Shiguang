## 2. 目录职责说明

### 2.1 根目录 `拾光计划/`

| 文件/目录 | 用途 |
| --- | --- |
| `README.md` | 项目概述、启动方式、技术栈说明 |
| `docs/` | 全部文档：规划、SQL 设计、目录规划 |
| `backend/` | 后端工程，独立 Maven 项目，可单独用 IDE 打开 |
| `frontend/` | 前端工程，独立 npm 项目，可单独用 VS Code 打开 |
| `.gitignore` | 忽略 node_modules、target、.idea 等 |

### 2.2 后端 `backend/`

| 层级 | 目录 | 职责 |
| --- | --- | --- |
| 启动 | `ShiguangApplication.java` | Spring Boot 启动入口 |
| 公共 | `common/` | 统一返回体、全局配置、工具类 |
| 模型 | `model/domain/` | 数据库实体类（对应三张表） |
| 传输 | `model/dto/` | 前后端交互用的数据传输对象 |
| 持久层 | `mapper/` | MyBatis-Plus Mapper 接口 |
| 业务层 | `service/` | 接口 + 实现，核心业务逻辑 |
| 控制层 | `web/controller/` | REST 接口，接收请求、调用 Service、返回 JsonResponse |
| 配置 | `resources/application.yml` | OceanBase 连接、端口、文件上传等 |
| SQL | `resources/mapper/` | 复杂 SQL 的 XML 映射（简单查询走 MyBatis-Plus 内置方法） |

### 2.3 前端 `frontend/`

| 目录 | 职责 |
| --- | --- |
| `api/` | 按模块拆分的接口调用函数，每个文件对应一个后端 Controller |
| `router/` | 路由定义 + 路由守卫（未登录跳转、角色权限） |
| `stores/` | Pinia 状态管理（用户信息、登录态） |
| `utils/` | Axios 封装，统一处理 baseURL、超时、拦截器 |
| `assets/` | 图片、全局 SCSS |
| `views/` | 按功能模块分文件夹，每个文件夹内放相关页面 |

## 3. 前后端通信约定

### 3.1 接口前缀

后端所有 API 统一以 `/api` 开头：

```
http://localhost:8080/api/auth/login
http://localhost:8080/api/goal/list
http://localhost:8080/api/checkin/add
...
```

### 3.2 跨域处理

开发阶段前端运行在 `localhost:5173`（Vite 默认），后端运行在 `localhost:8080`。

两种处理方式：
- **方案 A（推荐）**：Vite 配置代理，`vite.config.js` 中将 `/api` 转发到 `localhost:8080`
- **方案 B**：后端 `WebConfig.java` 配置 CORS 放行

建议两种都配上，确保本地开发和后续部署都不出问题。

### 3.3 响应格式

所有后端接口统一返回：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

## 4. 初始化步骤建议

1. 在 `拾光计划/` 下用 `mkdir backend frontend` 建好两个子目录
2. `backend/` 下用 IDE（如 IntelliJ）创建 Spring Boot 项目，包名 `com.example.shiguang`
3. `frontend/` 下用 `npm create vite@latest . -- --template vue` 初始化，再装 Element Plus 等依赖
4. 根目录配 `.gitignore`，确保不提交 `node_modules`、`target`、`.idea`
5. 按本文目录结构逐步补全文件
6. 全程在 `拾光计划/` 根目录执行 Git 操作，一次提交包含前后端代码
