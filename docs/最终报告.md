## 一、项目概况

拾光计划是一个面向学生和个人学习者的**学习打卡与目标管理系统**，采用前后端分离架构，涵盖目标设定、每日打卡、数据统计分析、成就徽章、标签分类、AI 智能助手等完整功能。

| 基本信息 | |
|----------|------|
| 项目名称 | 拾光计划 |
| 架构模式 | 前后端分离（Vue 3 + Spring Boot） |
| 数据库 | OceanBase（MySQL 兼容模式），utf8mb4 |
| 仓库地址 | https://github.com/JerryK988156/Project_Shiguang |
| 总文件数 | 约 96 个源码文件（不含 node_modules / target / dist） |

## 四、数据库设计

### 4.1 表结构总览（6 张表）

| 表名 | 说明 | 核心字段 | 约束 |
|------|------|----------|------|
| `user` | 用户表 | id, username(UNIQUE), password(BCrypt), nickname, role, avatar, profile, status | 账号唯一 |
| `goal` | 学习目标表 | id, user_id(FK), title, description, start_date, end_date, target_days, status | 普通索引 |
| `goal_tag` | 目标标签表 | id, goal_id(FK), tag_name | (goal_id, tag_name) 联合唯一 |
| `checkin_record` | 打卡记录表 | id, user_id(FK), goal_id(FK), checkin_date, study_duration, content, remark | (user_id, goal_id, checkin_date) 联合唯一 |
| `achievement` | 成就记录表 | id, user_id(FK), goal_id, goal_title, milestone_days, badge_name, achieved_date | (user_id, goal_id, milestone_days) 联合唯一 |
| `agent_message` | AI 聊天历史 | id, user_id, session_id, role, content | 普通索引 |
| `audit_log` | 审计日志 | id, user_id, action, target, ip | 普通索引 |

### 4.2 关键约束

- **`checkin_record.uk_user_goal_date`**：`(user_id, goal_id, checkin_date)` 联合唯一索引，从数据库层面确保"每任务每天只能打卡一次"，配合业务层考前校验，双保险。
- **`user.uk_username`**：用户名唯一，注册和修改资料时均有校验。
- **`achievement.uk_user_goal_milestone`**：同一用户同一目标不会重复获得同一级别成就。
- **`goal_tag.uk_goal_tag`**：同一目标不会重复绑定同名标签。

## 八、项目亮点

### 8.1 架构设计

1. **前后端分离规范**：独立 frontend/backend 目录，清晰的 api/router/stores/views 分层，后端 controller/service/mapper/model 标准分层。
2. **JWT 双 Token 机制**：access token（2h）+ refresh token（7d），前端 Axios 拦截器自动刷新，请求队列避免并发刷新。
3. **Redis 缓存策略**：统计接口全部 `@Cacheable`（TTL 5 分钟），写操作通过 `@CacheEvict` 联动清除，保证数据一致性。
4. **三级权限体系**：超级管理员(Jerry/id=1) > 管理员 > 普通用户，后端 + 前端路由双重校验。

### 8.2 工程化能力

5. **Docker 一键部署**：docker-compose.yml 编排 Redis + 后端 + 前端，支持环境变量驱动配置。
6. **数据库自检与自动初始化**：启动时自动验证数据库连接，首次启动自动建库建表 + 导入测试数据（5 用户 + 6 目标 + 60+ 条打卡记录）。
7. **OpenAPI 3.0 文档**：SpringDoc + Swagger UI，标注了 Bearer JWT 认证，所有 DTO 和端点均有文档。
8. **AOP 审计日志**：自定义 `@Auditable` 注解，自动记录关键操作（登录、删除目标、修改状态等）+ 客户端 IP。

### 8.3 业务功能

9. **AI 智能助手**：基于 LLM Function Calling，支持 8 种本地工具函数（创建目标、打卡、查询统计等），右下角浮动聊天窗口支持拖拽缩放和流式 SSE 输出。
10. **成就徽章系统**：6 级里程碑（7/14/21/30/60/100 天），打卡触发时自动检测并弹窗祝贺（canvas-confetti 动画）。
11. **打卡日历热力图**：GitHub 风格颜色映射，按用户目标数量动态设置 max 上限。
12. **学习周报**：本周 vs 上周对比（打卡天数、学习时长、活跃目标数），百分比变化展示。
13. **标签分类统计**：目标支持多标签，按标签汇总学习时长，目标列表支持标签筛选。

### 8.4 用户体验

14. **登录/注册合一页面**：毛玻璃渐变背景，模式切换流畅。
15. **首页看板数据动画**：数字从零缓入计数（requestAnimationFrame ease-out cubic）。
16. **目标模板库**：10 个预设模板（背单词、刷题、运动等），一键填充。
17. **CSV 数据导出**：打卡记录支持导出为 UTF-8 BOM CSV 文件（正确处理逗号/引号转义）。
18. **浏览器通知提醒**：Notification API，每天首次进入页面弹提醒，localStorage 防重复。
19. **骨架屏加载**：数据加载中显示 SkeletonCard 占位。
20. **Token 错误重试**：Axios 响应拦截器最多重试 2 次，防止闪断导致白屏。
