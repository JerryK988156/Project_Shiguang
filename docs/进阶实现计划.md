# 拾光计划 — 进阶实现计划

## 概述

本文档在现有 MVP 基础上，规划三个阶段的进阶功能。每个条目包含目标、实现方案和涉及文件清单。

---

## 第一阶段：体验优化（预计 3-5 小时）

### 1.1 首页看板增加打卡日历热力图

**目标**：Dashboard 底部增加与 PersonalStat 相同的日历热力图，让用户登录即看到整月打卡分布。

**实现方案**：
- Dashboard/Index.vue 引入 `getCheckinCalendarApi`
- 复用 PersonalStat 中 `renderCalendar` 的逻辑，新增一个 `calendarRef` 容器
- 在四格卡片和饼图之间插入日历卡片

**涉及文件**：
- `frontend/src/views/dashboard/Index.vue`（新增日历区块）

---

### 1.2 首页看板增加 30 天趋势切换

**目标**：Dashboard 中"近 7 天趋势"卡片增加一个 `el-radio-group` 切换按钮，支持在 7 天和 30 天之间切换。

**实现方案**：
- 新增 `trendRange` ref（'7' / '30'）
- 加载数据时根据 `trendRange` 调用 `getStatTrend7Api()` 或 `getStatTrend30Api()`
- `watch(trendRange)` 触发重新加载
- 30 天时自动切换为柱状图 + 折线图（与 PersonalStat 保持一致）

**涉及文件**：
- `frontend/src/views/dashboard/Index.vue`

---

### 1.3 目标列表按创建时间排序 + 分页

**目标**：目标列表当前按数据库返回顺序展示。增加按创建时间倒序排序和前端分页。

**实现方案**：
- 后端 `GoalController.list()` 增加 `@RequestParam(defaultValue = "createTime") String orderBy`
- `GoalService.list()` 增加排序逻辑：`wrapper.orderByDesc(orderBy)`
- 前端检查返回数据量，超过 10 条时显示 `el-pagination`

**涉及文件**：
- `backend/.../GoalController.java`（增加参数）
- `backend/.../GoalService.java`（增加排序逻辑）
- `frontend/src/views/goal/GoalList.vue`（接入分页组件）

---

### 1.4 打卡成功后成就提示

**目标**：用户完成打卡后，如果连续打卡达到特定天数（7/30/100），弹出一个 Achievement 卡片提示。

**实现方案**：
- 在 `CheckinAdd.vue` 的 `handleSubmit` 成功后，调用 `getStatOverviewApi()` 获取 `currentStreak`
- 检查是否命中里程碑（7 / 14 / 21 / 30 / 60 / 100）
- 命中时弹出 `el-dialog` 展示成就文案 + 动态天数
- 不命中则仅显示普通 "打卡成功"

**涉及文件**：
- `frontend/src/views/checkin/CheckinAdd.vue`（成就检测 + 弹窗）

---

### 1.5 统一 HTTP 方法为 RESTful

**目标**：将当前后端统一使用 POST 的接口改为规划文档要求的 RESTful 风格。

**实现方案**：
| 接口 | 当前方法 | 目标方法 |
| --- | --- | --- |
| `/api/goal/update` | POST | PUT |
| `/api/goal/delete/{id}` | POST | DELETE |
| `/api/goal/status/{id}` | POST | PATCH |
| `/api/checkin/delete/{id}` | POST | DELETE |
| `/api/user/profile` | POST | PUT |
| `/api/user/password` | POST | PUT |

后端改注解（`@PostMapping` → `@PutMapping` / `@DeleteMapping`），前端同步改 `method` 字段。

**涉及文件**：
- `backend/.../GoalController.java`
- `backend/.../CheckinController.java`
- `backend/.../UserController.java`
- `frontend/src/api/goal.js`
- `frontend/src/api/checkin.js`
- `frontend/src/api/auth.js`

---

### 1.6 优化热力图日历颜色映射

**目标**：当前视觉映射对所有用户一致。改为根据用户的目标数量动态设置 max 上限。

**实现方案**：
- 后端 `checkinCalendar()` 增加返回 `maxPossible` 字段（用户当前进行中的目标数）
- 前端用 `maxPossible` 作为 visualMap.max，使颜色分布在用户实际可能范围

**涉及文件**：
- `backend/.../StatService.java`（增加 maxPossible 计算）
- `frontend/src/views/stat/PersonalStat.vue`（使用新字段）
- `frontend/src/views/dashboard/Index.vue`（同步修改）

## 第二阶段：功能增强（预计 1-2 天）

### 2.1 用户注册功能

**目标**：支持新用户自主注册，填写账号/密码/昵称后即创建用户。

**实现方案**：
- 新建 `POST /api/auth/register` 接口，接收 `{ username, password, nickname }`
- `AuthService.register()` 中校验：
  - 用户名 4-20 位，仅字母数字下划线
  - 密码 ≥6 位
  - 用户名唯一性检查
  - MD5 加密后写入
- 新增 `RegisterDTO`
- 前端 Login.vue 增加"注册账号"链接，点击切换为注册表单
- 注册成功后自动登录并跳转 dashboard

**涉及文件**：
- `backend/.../model/dto/RegisterDTO.java`（新建）
- `backend/.../service/AuthService.java`（新增 register 方法）
- `backend/.../controller/AuthController.java`（新增 /register 端点）
- `frontend/src/views/Login.vue`（增加注册模式切换）
- `frontend/src/api/auth.js`（增加 registerApi）

### 2.2 打卡成就徽章系统

**目标**：用户个人中心展示已获得的成就徽章，如"坚持 7 天""满月打卡""百天达人"等。

**实现方案**：

- 后端 `AchievementService` 提供：
  - `checkAndAward(userId)` — 检查当前 streak 是否达到新里程碑
  - `listByUser()` — 返回用户所有徽章

- 前端 UserProfile.vue 新增"我的成就"区域，用卡片 + 图标展示徽章列
- 里程碑设计：
  - 7 天：青铜 🥉
  - 14 天：白银 🥈
  - 21 天：黄金 🥇
  - 30 天：钻石 💎
  - 60 天：大师 🏆
  - 100 天：传奇 👑

**涉及文件**：
- `backend/.../model/domain/Achievement.java`（新建实体）
- `backend/.../mapper/AchievementMapper.java`（新建）
- `backend/.../service/AchievementService.java`（新建）
- `backend/.../controller/AchievementController.java`（新建）
- `backend/src/main/resources/sql/bootstrap.sql`（新增建表）
- `frontend/src/views/user/UserProfile.vue`（新增成就区块）
- `frontend/src/api/achievement.js`（新建）

### 2.4 响应式移动端适配

## 第三阶段：进阶能力（可选，3-5 天）

### 3.1 JWT 鉴权替换 Session

**目标**：将当前 Session 登录机制替换为 JWT Token，支持无状态横向扩展。

**实现方案**：
- 引入 `jjwt` 依赖
- 新建 `JwtUtils` 工具类：`generateToken(userId, role)` / `parseToken(token)`
- 新建 `JwtInterceptor` 拦截器，从 `Authorization: Bearer xxx` 头解析 token
- 新建 `WebMvcConfig` 注册拦截器，排除 `/api/auth/login` `/api/auth/register`
- 移除 `SessionUtils` 中所有 HttpSession 操作
- 前端 `request.js` 增加 `Authorization` 请求头
- 前端 `stores/user.js` 将 token 从 localStorage mock 改为真实 JWT

**涉及文件**：
- `backend/pom.xml`（新增 jjwt 依赖）
- `backend/.../common/utls/JwtUtils.java`（新建）
- `backend/.../common/config/JwtInterceptor.java`（新建）
- `backend/.../common/config/WebMvcConfig.java`（修改拦截器注册）
- `backend/.../service/AuthService.java`（改用 JWT 签发）
- `frontend/src/utils/request.js`（增加 Authorization 头）
- `frontend/src/stores/user.js`（适配 JWT）

**实现方案**：

**涉及文件**：

### 3.4 学习周报自动生成

**目标**：每周一自动生成上一周的学习周报，展示打卡天数、总时长、完成目标、对比上周变化。

**实现方案**：
- 后端新增 `GET /api/stat/weekly_report` 接口，返回：
  - `totalDays`：上周打卡天数
  - `totalMinutes`：上周总时长
  - `completedGoals`：上周完成的目标
  - `vsLastWeek`：各指标环比变化百分比
- 前端 PersonalStat 新增"周报"区块，展示本周 vs 上周对比卡片
- 可选：增加分享按钮，生成一张包含周报数据的图片（html2canvas）

**涉及文件**：
- `backend/.../service/StatService.java`（新增 weeklyReport 方法）
- `backend/.../controller/StatController.java`（新增端点）
- `frontend/src/views/stat/PersonalStat.vue`（新增周报区块）
- `frontend/src/api/stat.js`（新增 API）
